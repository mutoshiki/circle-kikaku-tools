# ミニマルSaaS UI刷新レポート

## 目的

既存の機能、情報構造、主要DOM、画面構成、要素の並び、操作方法を維持しながら、表示だけを単一のミニマルSaaSデザインへ刷新した。
例外として、依頼に従い外観切替機能に属するメニュー、モーダル、状態管理、保存処理、専用CSS、テスト資産は完全に削除した。

## デザイン方針

- 淡いニュートラル背景と白い主要表面
- 濃いネイビー系本文と控えめな補助文字
- 主操作だけに青いアクセントを使用
- 角丸を7〜12pxへ整理
- 浮遊感の強い影を減らし、細い境界と小さな影で階層化
- ボタン、入力、タブ、モーダル、カードの状態表現を統一
- 精算区分や学年など、意味を持つ色は低彩度で維持
- モバイルとデスクトップで同じ製品表現を維持

## 外観切替機能の削除

削除したもの：

- ヘッダーメニューの外観設定項目
- 外観設定モーダル
- HTMLの表示モード属性
- OS設定との連動処理
- localStorageの表示モード保存キー
- `assets/js/features/appearance.js`
- `assets/js/modules/theme-presets.js`
- `assets/css/theme/`一式
- `assets/css/settlement/dark/`一式
- 表示モード別の条件付きCSS
- 旧表示モード専用テスト、画像基準、監査ツール
- 旧機能を説明していた設計資料

`index.html`と`assets/`を対象に検索し、旧機能の属性、ID、保存キー、切替APIが残っていないことを確認した。

## 主な変更

### デザイントークン

`assets/css/tokens/01-color-scheme.css`を単一の色体系へ再構成した。背景、文字、境界、hover、意味色、影を意味トークンとして集約した。

`assets/css/tokens/03-bootstrap-controls.css`と`05-control-surface-tokens.css`で、操作部品の高さ、角丸、境界、フォーカス、主操作色を統一した。

### 共通の視覚表現

`assets/css/visual/01-saas-skin.css`を追加し、配置やグリッドを変更せずに、次の表面処理を統一した。

- ヘッダーとタブ
- ボタンと入力
- ドロップダウン
- モーダル
- 車・班・参加者カード
- 未割当トレイ
- 発表ビュー
- 精算カード、集金リスト、編集画面
- 空状態、通知、フォーカス状態

### owner CSS

旧表示モードの条件付きルールを各ownerから除去し、単一トークンを参照するルールへ整理した。レイアウト、DOM順序、画面遷移、入力方法は変更していない。

### モバイル

- 企画名の文字寸法を調整し、限られたヘッダー幅での可読性を改善
- 40〜44pxの操作領域を維持
- カードとモーダルの境界、余白、入力欄を統一
- 360pxでも文書全体と主要スクロール領域に横方向のはみ出しがないことを確認
