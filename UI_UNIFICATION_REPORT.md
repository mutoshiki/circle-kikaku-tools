# UI統一パス 実施メモ

## 洗い出した不統一
- 主要操作・補助操作・危険操作のボタン表現が場所によって混在していた。
- カード、入力欄、モーダル、タブで角丸と境界線の基準が少しずつ違っていた。
- テーマ変更時に、アクセント背景上の文字色が白固定になり、明るいアクセント色で読みにくい場合があった。
- Bootstrap系のドロップダウン、モーダル、入力欄に白背景・黒文字が残りやすい箇所があった。

## 変更内容
- `assets/css/00-base-tokens.css` の基本角丸を `--radius-main: 10px`、`--radius-sm: 8px` に整理。
- `assets/css/08-control-consistency.css` に最終UI統一パスを追加。
- 共通の余白、境界線、カード面、危険色、警告色のトークンを追加。
- ボタンを主要操作・補助操作・控えめ操作・危険操作に分けて再整理。
- 明るいアクセント色のテーマでは、主要ボタンの文字色を濃色に切り替えるようにした。
- カード、入力欄、モーダル、ドロップダウン、タブ周辺の角丸・境界線・文字色を統一。

## 確認
- JS、Firebase設定、ドラッグ処理、ロック処理、モーダル起動処理には触れていない。
- `npm test` の静的チェックを通過。

## 2026-05-11 テーマ変更時の主要ボタン文字色修正

- 原因: 明るいテーマカラーの可読性対策として、主要ボタンの文字色を `#111827` に切り替えるCSSを入れていたため、テーマ変更時に「共有」「参加者登録」「自動割り当て」などが黒文字になっていた。
- 修正: 主要ボタンの文字色は白に固定し、明るすぎるテーマだけボタン背景を少し暗くする方式に変更。
- 対象: warm / anthropic / claude / gruvbox / ayu / candy / skyblue / 8bit などの明るいアクセントテーマ。
- 確認: 静的テストに `theme-primary-text-fix-check.js` を追加し、主要ボタン文字色が黒に固定されないことを確認できるようにした。
