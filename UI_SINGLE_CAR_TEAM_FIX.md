# UI_SINGLE_CAR_TEAM_FIX

## 目的

車割と班を複数管理する仕様を廃止し、編集画面では「車割 / 班」をトグルで切り替えるだけにした。
参加者登録のメンバーは車割と班で分かれないように、共通の参加者リストとして同期する。

## 対応

1. 車割・班は各1つに固定
   - `plan-car` と `plan-team` の2つだけを保持する。
   - 既存データに複数の車割・班があっても、現在表示中の種類を優先しつつ各1つに正規化する。
   - UIから新規追加・複製・削除を取り除いた。

2. 参加者リストを共通化
   - 車割・班のどちらで参加者登録しても、もう一方にも同じ参加者が反映される。
   - 片方で追加した人は、もう片方では未割り当てとして追加される。
   - 片方で完全削除した人は、もう片方からも外れる。
   - 既存の配置はできるだけ維持する。

3. 編集画面のトグルは維持
   - 「車割 / 班」の切り替えはそのまま残した。
   - 表示中の名前と名前変更だけを残し、複数候補のチップや追加操作は削除した。

4. 発表ビューを横並びに戻す
   - 現在表示中の片方だけではなく、車割と班の両方を発表ビューに表示する。
   - 表示順は車割 → 班。
   - `#sheet-canvas` は横方向の並びに戻した。

## 確認

- `node --check` で全JS構文確認済み。
- `npm test` の静的テスト35件はすべて通過。
